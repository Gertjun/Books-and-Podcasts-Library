Setup Guide

Add books & podcasts directly from the website

This guide walks you through connecting your library to GitHub, so you can add entries from any device and have changes sync everywhere automatically.

✓ Files uploaded to GitHub ✓ GitHub Pages live Step 1 → Create a token Step 2 → Update your files Step 3 → Done
1

Create a GitHub personal access token

A personal access token (PAT) is a password the website uses to write changes back to your GitHub repository. You only need to create this once.

  1. Go to github.com and make sure you're logged in.
  2. Click your profile picture (top right) → Settings.
  3. Scroll down the left sidebar to Developer settings (very bottom).
  4. Click Personal access tokens → Fine-grained tokens.
  5. Click Generate new token.
  6. Give it a name, e.g. my-library-editor.
  7. Set an expiration — 1 year is a good balance of security and convenience.
  8. Under Repository access, choose Only select repositories and select your my-library repo.
  9. Under Permissions → Repository permissions, find Contents and set it to Read and write.
  10. Click Generate token at the bottom.
  11. Copy the token immediately — GitHub will only show it once. It looks like: github_pat_abc123...
Important: Treat this token like a password. Anyone who has it can edit your repository. Don't share it or commit it to GitHub.
Good news: Fine-grained tokens are scoped to only your my-library repo and only the Contents permission — so even if it were ever leaked, it can't access anything else in your GitHub account.
2

Download the updated files from Claude

Ask Claude to build the updated versions of index.html and source.html with the "Add book" feature and GitHub API integration built in. You'll need to provide three things:

  1. Your GitHub username (visible in the URL when you're on github.com, e.g. github.com/yourusername).
  2. Your repository name (e.g. my-library).
  3. Your personal access token from Step 1.

You can say something like:

Tell Claude: "Please build the Add Book feature for my library. My GitHub username is: yourusername My repository name is: my-library My personal access token is: github_pat_abc123..."

Claude will give you updated versions of index.html and source.html with the form and GitHub API integration included.

3

Upload the updated files to GitHub

Replace the existing files in your repository with the new ones Claude gives you.

  1. Go to your repository on github.com.
  2. Click on index.html in the file list.
  3. Click the pencil icon (Edit this file) in the top right of the file view.
  4. Select all the existing content and delete it, then paste in the new content from Claude.
  5. Click Commit changes.
  6. Repeat for source.html.

Alternatively, you can upload the files directly: on the repository home page, click Add file → Upload files, drop in both updated files, and commit.

Note: GitHub Pages may take up to a minute to reflect the changes. If you don't see the update immediately, wait 60 seconds and refresh.
4

Try it out

Open your GitHub Pages URL on your iPhone (or any browser), tap Add book, fill in the details, and save. The page will commit the change directly to your data.js file in GitHub — no computer needed.

  1. Open yourusername.github.io/my-library in Safari.
  2. Tap Add book.
  3. Fill in the title, type (book or podcast), and any notes.
  4. Tap Save. The page updates automatically.
Tip: Add the page to your iPhone Home Screen for one-tap access. In Safari, tap the share icon → Add to Home Screen.

What you'll have when this is done

  • An "Add book" button on the library home page
  • A form to enter title, type, key ideas, quotes, and episodes
  • Saves write directly to GitHub — synced everywhere instantly
  • Works on iPhone, iPad, and desktop — no app needed
  • Token expires in 1 year — just regenerate it when it does
?

When your token expires

GitHub will email you before the token expires. When it does, go back to Developer settings → Fine-grained tokens, find your token, and click Regenerate. Then ask Claude to update the token in your files and re-upload them to GitHub.
